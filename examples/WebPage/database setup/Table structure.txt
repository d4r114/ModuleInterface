Database should contain two tables:

settings:
id (char[6], primary key)
value (TINYTEXT, allow NULL)
description (TINYTEXT, allow NULL)

timeseries, mandatory columns:
id (BIGINT, AUTO_INCREMENT)
time (DATETIME, key, default CURRENT_TIMESTAMP)
scan1m (TINYINT, allow NULL)
scan10m (TINYINT, allow NULL)
scan1h (TINYINT, allow NULL)
scan1d (TINYINT, allow NULL)

timeseries, one column for each output from the modules, for example:
smT1 (FLOAT, allow NULL) (sm=SensorMonitor, T=Temperature)

NOTE: ALL outputs from ALL the modules must have an associated column
in the timeseries table, otherwise the transfer will fail.
